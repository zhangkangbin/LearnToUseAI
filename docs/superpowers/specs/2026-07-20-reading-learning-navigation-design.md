# 阅读与学习导航设计

## 目标

在仓库根目录增加一篇统一导航文章，让不同基础、不同目标的读者都能快速确定学习起点、阅读顺序和实践方向。

## 设计方案

采用“递进主线 + 快捷路线”的双层结构：

- 递进主线按“基础篇 → 进阶篇 → 精通篇 → AI 能力系统 → 高级 AI 系统”展开，适合希望系统学习的读者。
- 快捷路线按学习目标组合现有文章，覆盖职场提效、自动化开发、RAG 知识库、生产可靠性与安全、多智能体架构五类读者。
- 每个阶段都说明学习目标、推荐输出和完成标准，避免只有链接而没有行动指引。
- 用一个贯穿项目“项目周报助手”把提示词、Skill、MCP、函数调用、RAG、评估和安全等主题串起来。

## 内容结构

1. 导航使用方法与路线总览图。
2. 五阶段递进主线，链接到全部核心入口及专题文章。
3. 五条快捷路线，给出可直接点击的精确阅读顺序。
4. 高级 AI 系统四阶段导航，覆盖 01—28 篇文章。
5. 单篇文章学习法、贯穿实践项目、完成标准和常见误区。
6. 按读者身份给出“今天从哪里开始”的入口。

## 验收标准

- 根目录存在 `阅读与学习导航.md`。
- 基础、进阶、精通、AI 能力系统和高级 AI 系统均有清晰入口。
- 高级 AI 系统 28 个专题均能通过导航直接或经分阶段索引访问。
- 至少包含一张 Mermaid 路线图。
- 文章内所有相对链接都指向现有文件。
- Markdown 代码围栏成对闭合。
