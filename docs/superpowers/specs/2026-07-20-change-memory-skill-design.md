# 修改记录长期记忆与总结 Skill 设计

## 目标

为仓库建立可版本控制、可持续追加的修改长期记忆，并提供一个可复用 Skill，在每次完成一组有意义的仓库修改后，根据 Git 差异生成中文总结，记录精确到年月日时分的北京时间。

## 记录粒度

“每次修改”定义为一个任务内完成的一组相关变更，而不是编辑器的每次保存动作。一次任务即使修改多个文件，也只生成一条记录；不同目标的任务分别记录。

以下情况需要记录：

- 新增、修改、重命名或删除仓库文件。
- 完成文档、代码、配置或 Skill 的一组变更。
- 用户明确要求总结、记录变更、更新修改日志或写入长期记忆。

以下情况不单独记录：

- 只读检查、搜索、解释或状态查询。
- 仅为了追加 `修改记录.md` 而产生的文件变化。
- 同一任务中对尚未定稿内容的临时保存。

## 文件结构

### `修改记录.md`

仓库根目录的长期记忆文件，按时间正序追加记录。每条记录包含：

```markdown
## 2026年07月20日 11:13｜新增阅读与学习导航

- **类型**：文档
- **修改摘要**：
  - 新增递进主线与五条快捷学习路线。
- **涉及文件**：
  - `阅读与学习导航.md`
- **验证结果**：
  - 37 个目标文章链接全部覆盖。
- **来源**：Git 提交 `c0d47b3`（可选）
```

时间格式固定为 `yyyy年MM月dd日 HH:mm`，时区固定为 `Asia/Shanghai`。历史补录可以填写 Git 提交短哈希；当前未提交变更不强求提交哈希，避免提交哈希自引用问题。

### `AGENTS.md`

根目录仓库规则，要求未来代理在完成任何写入型任务后、提交或最终答复前使用 `recording-change-memory` Skill。规则明确：

- 每个任务只记录一次。
- 从 Git 工作区和暂存区读取真实差异，不凭记忆编造。
- 不把 `修改记录.md` 自身计入修改摘要。
- 记录与业务变更进入同一个提交。

### `skills/recording-change-memory/SKILL.md`

仓库内的 Skill 源码。Skill 的触发场景覆盖“完成仓库修改”“总结本次变更”“更新修改记录”“写入长期记忆”“提交前记录”等表达。

Skill 负责需要判断的工作：

1. 确认仓库根目录和记录文件。
2. 读取 `git status --short`、未暂存差异、已暂存差异和必要的未跟踪文件内容。
3. 排除记录文件自身，基于证据生成简短中文标题、类型、摘要、文件列表和验证结果。
4. 调用确定性脚本追加记录。
5. 校验时间格式、记录唯一性和文件状态。

### `skills/recording-change-memory/scripts/add-change-record.ps1`

脚本只处理机械且容易出错的部分：

- 将当前 UTC 时间转换为北京时间。
- 生成精确到分钟的固定格式时间戳。
- 对标题、摘要、文件和验证信息做非空校验。
- 以 UTF-8 编码追加 Markdown 条目。
- 如果同一分钟、同一标题的记录已存在，则拒绝重复追加。
- 支持显式传入时间，供历史补录和自动化测试使用。

脚本不自行总结 Git 差异，确保中文语义总结仍由 Skill 完成。

### `skills/recording-change-memory/agents/openai.yaml`

提供 Codex UI 元数据：中文显示名称、简短说明和默认调用提示。该文件由官方 Skill 初始化工具生成或校验，不手工添加无关字段。

### 本机安装目录

仓库版本完成验证后，将同一 Skill 安装到 `C:\Users\admin\.codex\skills\recording-change-memory`。仓库内版本是可提交、可共享的源，个人 Skills 目录是 Codex 自动发现的安装副本。

## 数据流

1. 用户或仓库规则触发 Skill。
2. Skill 从 Git 获取当前任务的真实变更证据。
3. Skill 排除 `修改记录.md`，形成中文修改总结。
4. PowerShell 脚本生成北京时间并追加标准条目。
5. Skill 重新读取记录，检查条目、链接、编码和工作区状态。
6. 记录文件与本次业务修改一起提交。

## 历史补录

首次创建 `修改记录.md` 时，从现有非合并 Git 提交补录重要修改。当前仓库至少补录：

- 初始 AI 基础、进阶和精通文章。
- AI 能力系统与高级 AI 系列文章。
- 阅读与学习导航文章。

历史记录使用提交时间转换后的北京时间，精确到分钟，并注明来源提交。纯合并提交不重复生成业务记录。

## 异常处理

- **不在 Git 仓库中**：停止并说明无法可靠确定变更范围。
- **没有业务变更**：不创建空记录，并报告无需记录。
- **记录文件不存在**：在仓库根目录按固定标题和说明创建。
- **Git 差异过大**：先按目录和变更类型归类，再形成 3—7 条高价值摘要。
- **脚本参数为空**：返回非零退出码，不写文件。
- **重复记录**：返回非零退出码并保留原文件不变。
- **验证失败**：不宣称记录成功，也不继续提交或推送。

## 测试设计

### RED：无 Skill 基线

让一个没有读取新 Skill 的代理处理“完成修改后写入长期记录”的场景，观察它是否会遗漏固定时区、统一格式、Git 证据、去重或递归保护，记录原始行为作为基线。

### GREEN：使用 Skill

让新的代理读取 Skill 后处理相同场景，验证它能：

- 从 Git 差异而非对话记忆生成总结。
- 排除 `修改记录.md` 自身。
- 使用固定北京时间格式。
- 调用脚本而不是手工拼接易错格式。
- 在无业务变更时拒绝创建空记录。

### 脚本验证

在临时 Git 仓库或临时目录验证：

- 指定时间能生成 `yyyy年MM月dd日 HH:mm` 格式。
- 中文标题、摘要和文件名保持 UTF-8。
- 同一分钟、同一标题第二次调用失败且文件不变。
- 缺少标题、摘要、文件或验证信息时不写入。

### 仓库验证

- `quick_validate.py` 验证 Skill 元数据和目录命名。
- `agents/openai.yaml` 与 `SKILL.md` 保持一致。
- `修改记录.md` Markdown 围栏闭合、时间格式一致。
- 根目录文章的现有相对链接继续有效。
- `git diff --check` 无空白错误。

## 验收标准

- 根目录存在中文文件名 `修改记录.md`，包含历史补录和本次功能记录。
- 所有记录时间均使用北京时间，精确到年月日时分。
- 仓库内存在可验证的 `recording-change-memory` Skill 和确定性追加脚本。
- 本机 Codex Skills 目录安装相同版本。
- `AGENTS.md` 能让未来写入型任务在结束前触发记录流程。
- Skill 的基线测试、使用后测试、脚本测试和格式验证均有新鲜证据。
- 不会因修改记录文件自身产生无限递归记录。
