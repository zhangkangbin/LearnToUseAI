## 建议的实践项目

用一个“项目周报助手”贯穿全系列：先定义上下文和结构化输出，再接入项目资料；随后增加状态、评估、日志、审批和安全控制；最后尝试多智能体、后台任务和治理机制。这样每学一篇，都能在同一个项目上增加一个可验证能力。

## 学习原则

- 先做确定性 Workflow，再增加 Agent 自主性；
- 先接只读数据，再考虑写操作；
- 先建立评估与日志，再扩大使用范围；
- 先解决输入、流程和工具问题，再考虑微调模型；
- 所有高影响操作都要有明确责任人和人工确认。
